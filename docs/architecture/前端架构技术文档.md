## 一、项目概述

### 1.1 定位

本前端项目是俄罗斯电商AI自动化运营系统的Web管理端，为卖家提供三平台（Wildberries / Ozon / Yandex Market）统一运营仪表盘。支持PC浏览器和企业微信内嵌H5访问。

### 1.2 核心功能规划

| 模块 | 页面 | 路由 | 状态 |
|------|------|------|------|
| 认证 | 登录/注册 | `/login` | 已完成 |
| 首页 | 大盘总览 | `/` | 已完成（骨架+健康检查） |
| 广告 | 广告管理 | `/ads` | 占位 |
| 商品 | 商品管理 | `/products` | 占位 |
| SEO | SEO优化 | `/seo` | 占位 |
| 报表 | 数据报表 | `/reports` | 占位 |
| 设置 | 系统设置 | `/settings` | 占位 |

## 四、核心架构详解

### 4.1 构建配置 (`vite.config.js`)

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),  // 路径别名
    },
  },
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://47.84.130.136',       // 代理到后端服务器
        changeOrigin: true,
      },
    },
  },
  build: {
    outDir: 'dist',
    sourcemap: false,
  },
})
```

**关键配置说明：**

| 配置项 | 值 | 说明 |
|--------|---|------|
| `resolve.alias.@` | `./src` | 绝对路径导入，避免 `../../` 相对路径地狱 |
| `server.port` | 3000 | 开发服务器端口 |
| `server.proxy./api` | `http://47.84.130.136` | 开发环境API代理，解决跨域 |
| `build.outDir` | `dist` | 生产构建输出目录 |
| `build.sourcemap` | false | 生产环境不生成sourcemap，保护源码 |

**开发/生产环境API访问路径：**

```
开发环境:
  浏览器 → localhost:3000/api/v1/xxx
       → Vite DevServer Proxy
       → http://47.84.130.136/api/v1/xxx

生产环境:
  浏览器 → 47.84.130.136/api/v1/xxx
       → Nginx反向代理
       → uvicorn:8000/api/v1/xxx
```

### 4.2 环境变量

## 五、API请求层详解 (`src/api/`)

### 5.1 Axios统一封装 (`api/index.js`)

整个前端项目的HTTP请求通过唯一的Axios实例 `request` 发出，任何组件不得直接使用 `axios` 或 `fetch`。

**实例配置：**

```javascript
const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,  // '/api/v1'
  timeout: 15000,                                // 15秒超时
  headers: { 'Content-Type': 'application/json' },
})
```

**请求拦截器 — 自动携带JWT：**

```javascript
request.interceptors.request.use((config) => {
  const token = useAuthStore.getState().token
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

- 通过 `useAuthStore.getState()` 在非React组件中读取 Zustand store
- 每次请求自动注入 `Authorization: Bearer <token>` 请求头
- 未登录时（token为null）不添加该header

**响应拦截器 — 统一错误处理：**

```javascript
request.interceptors.response.use(
  (response) => {
    const res = response.data
    if (res.code !== 0) {
      // 认证类错误码：自动登出
      if (res.code === 20001 || res.code === 20002) {
        useAuthStore.getState().logout()
        window.location.href = '/login'
      }
      return Promise.reject(new Error(res.msg || '请求失败'))
    }
    return res  // 直接返回 {code, msg, data, timestamp}
  },
  (error) => {
    if (error.response) {
      if (status === 401 || status === 403) {
        useAuthStore.getState().logout()
        window.location.href = '/login'
      }
    }
    return Promise.reject(error)
  }
)
```

**后端统一响应格式适配：**

```
后端返回:  { code: 0, msg: "success", data: {...}, timestamp: 1775552000 }
                ↓ 响应拦截器处理
code === 0 → 直接返回整个对象，组件通过 res.data 获取业务数据
code !== 0 → 抛出 Error(msg)，组件在 catch 中展示错误信息
```

**错误码与前端处理对照：**

| 错误码 | 含义 | 前端处理 |
|--------|------|---------|
| 0 | 成功 | 正常返回数据 |
| 20001 | AUTH_FAILED（登录失败） | 清除token，跳转登录页 |
| 20002 | TOKEN_EXPIRED | 清除token，跳转登录页 |
| 30001 | SHOP_NOT_FOUND | 组件层catch展示提示 |
| 30003 | SHOP_LIMIT_EXCEEDED | 组件层catch展示提示 |
| 其他 | 业务错误 | 组件层catch展示msg |

## 六、状态管理详解 (`src/stores/`)

### 6.1 认证Store (`authStore.js`)

使用 Zustand + persist 中间件，管理全局登录状态。

```javascript
export const useAuthStore = create(
  persist(
    (set) => ({
      token: null,      // JWT令牌
      user: null,       // 用户信息 {id, tenant_id, username, email, role, status}
      tenant: null,     // 租户信息 {id, name, plan, max_shops}

      setAuth: (token, user, tenant) => set({ token, user, tenant }),
      logout: () => set({ token: null, user: null, tenant: null }),
    }),
    {
      name: 'auth-storage',           // localStorage key
      partialize: (state) => ({       // 只持久化这3个字段
        token: state.token,
        user: state.user,
        tenant: state.tenant,
      }),
    }
  )
)
```

**设计要点：**

| 特性 | 实现方式 | 说明 |
|------|---------|------|
| 持久化 | `persist` 中间件 + localStorage | 刷新页面不丢失登录态 |
| 选择性持久化 | `partialize` | 只存token/user/tenant，不存函数 |
| 非React访问 | `useAuthStore.getState()` | Axios拦截器中使用 |
| React组件订阅 | `useAuthStore((s) => s.token)` | 精准订阅，避免不必要re-render |

**状态流转：**

```
未登录状态: { token: null, user: null, tenant: null }
       ↓ 登录成功 → setAuth(token, user, tenant)
已登录状态: { token: "eyJ...", user: {...}, tenant: {...} }
       ↓ 退出/token失效 → logout()
未登录状态: { token: null, user: null, tenant: null }
       ↓ localStorage清空
       ↓ 重定向到 /login
```

## 七、路由与权限控制

### 7.1 路由架构 (`App.jsx`)

采用 React Router 6 的嵌套路由 + Outlet 模式：

```
<Routes>
  <Route path="/login" element={<Login />} />           ← 公开页面
  <Route path="/" element={<AuthRoute><AppLayout /></AuthRoute>}>  ← 需认证
    <Route index element={<Dashboard />} />             ← 首页大盘
    <Route path="ads" element={<Ads />} />
    <Route path="products" element={<Products />} />
    <Route path="seo" element={<Seo />} />
    <Route path="reports" element={<Reports />} />
    <Route path="settings" element={<Settings />} />
  </Route>
  <Route path="*" element={<Navigate to="/" />} />     ← 404兜底
</Routes>
```

**路由嵌套关系图：**

```
/login ─────────→ Login（全屏独立页面）

/ ──→ AuthRoute ──→ AppLayout
      │                 ├── Sider（侧边栏导航）
      │                 ├── Header（顶部栏）
      │                 └── Content → <Outlet />
      │                                  ├── / → Dashboard
      │                                  ├── /ads → Ads
      │                                  ├── /products → Products
      │                                  ├── /seo → Seo
      │                                  ├── /reports → Reports
      │                                  └── /settings → Settings
      │
      └── token不存在 → <Navigate to="/login" />

/* ──→ 重定向到 /
```

### 7.2 鉴权守卫 (`AuthRoute.jsx`)

```javascript
const AuthRoute = ({ children }) => {
  const token = useAuthStore((s) => s.token)
  if (!token) {
    return <Navigate to="/login" replace />
  }
  return children
}
```

**鉴权流程：**

1. 用户访问任何需认证路由（如 `/`、`/ads`）
2. `AuthRoute` 从 Zustand store 读取 token
3. token 存在 → 渲染子组件（AppLayout + 页面）
4. token 不存在 → 重定向到 `/login`，使用 `replace` 避免回退到无权页面

```
    │  Login.jsx → 调用auth API             │
    │  Dashboard.jsx → 调用system API       │
    │  Settings.jsx → 调用shops API (后续)  │
    └───────────────┬───────────────────────┘
                    │
                    ▼
    ┌───────────────────────────────────────┐
    │        Zustand Store (stores/)        │
    │                                       │
    │  authStore: token / user / tenant     │
    │      ↕ persist ↕                      │
    │  localStorage: auth-storage           │
    └───────────────────────────────────────┘
```

### 9.2 认证数据流

```
登录成功
    ↓
API返回 { access_token, user }
    ↓
authStore.setAuth(token, user, tenant)
    ↓
┌─────────────────────────────────┐
│ Zustand Store (内存)             │
│   token: "eyJ..."              │
│   user: { id:1, username:"张三"} │
│   tenant: null                  │
└─────────┬───────────────────────┘
          │ persist中间件自动同步
          ▼
┌─────────────────────────────────┐
│ localStorage                    │
│   key: "auth-storage"           │
│   value: JSON序列化的state       │
└─────────────────────────────────┘

后续请求:
  Axios请求拦截器 → useAuthStore.getState().token → 注入Header

页面刷新:
  Zustand persist → 从localStorage恢复state → token仍在 → 无需重新登录

退出登录:
  authStore.logout() → state归零 → localStorage清空 → 跳转/login
```

## 十、与后端对接规范

### 10.1 后端响应格式约定

所有后端接口统一返回：

```json
{
  "code": 0,           // 0=成功, 非0=错误
  "msg": "success",    // 说明文字
  "data": {},          // 业务数据
  "timestamp": 1775552000
}
```

前端响应拦截器在 `code !== 0` 时自动抛异常，组件只需处理 `res.data`。

### 10.2 分页数据格式约定

后端分页接口返回：

```json
{
  "items": [],          // 数据列表
  "total": 100,         // 总记录数
  "page": 1,            // 当前页码
  "page_size": 20,      // 每页条数
  "pages": 5            // 总页数
}
```

前端 Ant Design Table 的 `pagination` 直接对接：

```javascript
pagination={{
  current: data.page,
  pageSize: data.page_size,
  total: data.total,
  onChange: (page, pageSize) => fetchData({ page, page_size: pageSize })
}}
```

### 10.3 认证方式

- 方式：Bearer Token（JWT）
- Header：`Authorization: Bearer <access_token>`
- 有效期：24小时（86400秒）
- 刷新：调用 `POST /api/v1/auth/refresh` 获取新token

## 十三、后续开发指南

### 13.1 新增页面步骤

1. 在 `src/pages/` 创建页面组件（PascalCase命名）
2. 在 `src/App.jsx` 添加路由
3. 在 `src/components/AppLayout.jsx` 的 `menuItems` 添加菜单项
4. 如需API接口，在 `src/api/` 创建对应模块

### 13.2 新增API模块步骤

1. 在 `src/api/` 创建模块文件（如 `products.js`）
2. 导入统一的 `request` 实例
3. 按接口封装为独立函数并导出

```javascript
// src/api/products.js
import request from './index'

export function getProducts(params) {
  return request.get('/products', { params })
}
```

### 13.3 新增全局状态步骤

1. 在 `src/stores/` 创建 store 文件
2. 使用 `create` 创建 store
3. 需要持久化时使用 `persist` 中间件

### 13.4 编码规范

| 规范 | 说明 |
|------|------|
| 组件命名 | PascalCase（如 `ShopCard.jsx`） |
| 文件一个组件 | 每个文件只导出一个组件 |
| API调用 | 只通过 `src/api/` 模块，禁止组件内直接使用axios/fetch |
| 环境变量 | 禁止硬编码API地址，统一用 `VITE_API_BASE_URL` |
| 状态管理 | 全局状态用Zustand store，组件局部状态用useState |
| 样式 | 优先使用Ant Design内置样式/token，不写额外CSS文件 |

## 十四、已知问题与优化计划

| 优先级 | 问题 | 优化方案 |
|--------|------|---------|
| 中 | 单chunk过大（780KB） | 使用 `manualChunks` 拆分antd/echarts为独立chunk |
| 低 | 无代码分割 | 页面级组件改用 `React.lazy()` + `Suspense` 懒加载 |
| 低 | 无请求缓存 | 引入 `swr` 或 `react-query` 管理服务端状态 |
| 低 | token过期无自动刷新 | 拦截器中检测401，自动调用refresh后重试原请求 |
| 低 | 无国际化 | 后续如需俄语界面，引入 `react-i18next` |
