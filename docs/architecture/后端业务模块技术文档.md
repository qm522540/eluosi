## 二、认证模块详解

### 2.1 接口列表

| 接口 | 方法 | 认证 | 说明 |
|------|------|------|------|
| `/api/v1/auth/login` | POST | 否 | 邮箱+密码登录 |
| `/api/v1/auth/register` | POST | 否 | 注册（自动创建租户） |
| `/api/v1/auth/me` | GET | 是 | 获取当前用户+租户信息 |
| `/api/v1/auth/refresh` | POST | 是 | 刷新JWT令牌 |

### 2.2 认证流程

```
注册流程:
POST /auth/register { username, email, password, tenant_name }
    → 校验邮箱唯一性
    → 创建Tenant (plan=free, max_shops=3)
    → 创建User (role=owner, bcrypt加密密码)
    → 事务提交

登录流程:
POST /auth/login { email, password }
    → 查询User (email + status=active)
    → bcrypt校验密码
    → 生成JWT (payload: user_id, tenant_id, role)
    → 更新last_login_at
    → 返回 { access_token, user }

JWT认证链:
请求 → HTTPBearer提取token → jose.jwt.decode → 提取user_id/tenant_id
    → get_current_user依赖注入 → get_tenant_id提取租户ID
    → 所有查询自动过滤tenant_id（多租户隔离）
```

### 2.3 JWT配置

| 配置项 | 值 | 说明 |
|--------|---|------|
| 算法 | HS256 | 对称加密 |
| 有效期 | 1440分钟（24小时） | 环境变量 JWT_EXPIRE_MINUTES |
| 密钥 | 服务器.env自动生成 | 环境变量 JWT_SECRET_KEY |

### 2.4 密码加密

- 库：passlib + bcrypt 4.3.0
- 方案：bcrypt哈希（自动加盐）
- 注意：bcrypt 5.x与passlib 1.7.4不兼容，必须使用bcrypt < 5

## 三、店铺管理模块详解

### 3.1 接口列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/shops` | GET | 店铺列表（platform/status筛选+分页） |
| `/api/v1/shops` | POST | 创建店铺（校验数量限制） |
| `/api/v1/shops/{id}` | GET | 店铺详情（含凭证状态标记） |
| `/api/v1/shops/{id}` | PUT | 更新店铺（部分更新） |
| `/api/v1/shops/{id}` | DELETE | 软删除店铺 |
| `/api/v1/shops/{id}/test-connection` | POST | 测试平台API连接 |

### 3.2 三平台凭证

| 平台 | 必填凭证 | 可选凭证 |
|------|---------|---------|
| Wildberries | api_key | api_secret |
| Ozon | api_key + client_id | api_secret |
| Yandex Market | oauth_token | oauth_refresh_token |

### 3.3 安全设计

- 列表接口不返回任何凭证明文
- 详情接口只返回 `has_api_key`/`has_api_secret` 等布尔标记
- 凭证字段只能通过PUT接口更新

### 3.4 店铺数量限制

```
创建店铺时:
1. 查询租户的 max_shops 值（free=3, basic=10, pro=50, enterprise=无限）
2. 统计当前非deleted状态的店铺数
3. 超限返回错误码 30003 (SHOP_LIMIT_EXCEEDED)
```

### 5.2 设计原则

**广告活动不提供创建接口。** 广告活动由Celery定时任务从WB/Ozon/Yandex平台API同步而来，后端只提供查询和局部调整接口。

### 5.3 统计数据查询

`GET /api/v1/ads/stats?start_date=2026-04-01&end_date=2026-04-07`

**SQL聚合逻辑:**
```sql
SELECT stat_date, platform,
       SUM(impressions), SUM(clicks), SUM(spend),
       SUM(orders), SUM(revenue)
FROM ad_stats
WHERE tenant_id = ? AND stat_date BETWEEN ? AND ?
GROUP BY stat_date, platform
ORDER BY stat_date DESC
```

**计算指标:**

| 指标 | 公式 | 说明 |
|------|------|------|
| CTR | clicks / impressions × 100 | 点击率(%) |
| CPC | spend / clicks | 平均点击成本 |
| ACOS | spend / revenue × 100 | 广告销售成本率(%) |
| ROAS | revenue / spend | 广告投资回报率 |

### 5.4 汇总数据（Dashboard用）

## 七、企业微信通知服务详解

### 7.1 两种推送通道

| 通道 | 方式 | 使用场景 |
|------|------|---------|
| 群机器人Webhook | HTTP POST到Webhook URL | ROI告警、日报推送、任务失败 |
| 应用消息 | 企业微信API（需access_token） | 推送给指定用户 |

### 7.2 群机器人推送流程

```
send_wechat_work_bot(content, msg_type="markdown")
    → 检查 WECHAT_WORK_BOT_WEBHOOK 是否配置
    → 构建payload: { msgtype: "markdown", markdown: { content: "..." } }
    → POST到Webhook URL
    → 检查返回 errcode == 0
```

### 7.3 应用消息推送流程

```
send_wechat_work_app_message(user_ids, content)
    → 获取access_token:
        GET https://qyapi.weixin.qq.com/cgi-bin/gettoken
        ?corpid=xxx&corpsecret=xxx
    → 发送消息:
        POST https://qyapi.weixin.qq.com/cgi-bin/message/send
        ?access_token=xxx
        { touser: "user1|user2", msgtype: "text", agentid: xxx, text: { content: "..." } }
```

### 7.4 通知处理流程

```
系统产生通知:
  ROI告警任务 → INSERT INTO notifications (sent_at=NULL)
  日报生成任务 → INSERT INTO notifications (sent_at=NULL)
       ↓
process_pending_notifications():
  SELECT * FROM notifications WHERE sent_at IS NULL LIMIT 50
       ↓
  逐条调用 send_wechat_work_bot(markdown格式)
       ↓
  成功 → UPDATE sent_at = NOW()
  失败 → 不更新，下次重试
```

### 7.5 通知管理接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/notifications` | GET | 通知列表（支持is_read筛选） |
| `/api/v1/notifications/{id}/read` | PUT | 标记已读（更新is_read + read_at） |
| `/api/v1/notifications/send-pending` | POST | 手动触发发送（管理员操作） |

## 九、多租户隔离机制

### 9.1 隔离原理

```
JWT Token → { user_id: 1, tenant_id: 4, role: "owner" }
    ↓ dependencies.py
get_current_user() → 解析JWT提取tenant_id
    ↓
get_tenant_id() → 注入到路由函数
    ↓
所有service层查询:
  db.query(Model).filter(Model.tenant_id == tenant_id, ...)
```

### 9.2 所有数据表均包含tenant_id

21张表中，除系统表外的所有业务表都有 `tenant_id BIGINT NOT NULL INDEX` 字段。每次查询都过滤tenant_id，确保租户A看不到租户B的数据。

## 十、部署问题与修复记录

### 10.1 MySQL密码含@符号解析错误

**问题:** 数据库密码 `EcomApp@2026DB` 中的 `@` 被SQLAlchemy连接字符串解析为用户名与主机的分隔符。

**错误信息:**
```
Can't connect to MySQL server on '2026DB@localhost' ([Errno -2] Name or service not known)
```

**修复:** 在 `config.py` 中使用 `urllib.parse.quote_plus()` 对密码进行URL编码：
```python
from urllib.parse import quote_plus

@property
def DATABASE_URL(self) -> str:
    password = quote_plus(self.DB_PASSWORD)
    return f"mysql+pymysql://{self.DB_USER}:{password}@{self.DB_HOST}:{self.DB_PORT}/{self.DB_NAME}?charset=utf8mb4"
```

### 10.2 bcrypt版本不兼容

**问题:** `bcrypt 5.0.0` 移除了 `__about__` 模块，导致 `passlib 1.7.4` 加载失败。

**错误信息:**
```
AttributeError: module 'bcrypt' has no attribute '__about__'
```

**修复:** 降级bcrypt到4.x：
```bash
pip install 'bcrypt<5'  # 安装bcrypt 4.3.0
```
