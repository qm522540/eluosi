### 3.2 我的 review (commit `3c43dc9`, 全文见 `docs/daily/2026-05-04_review_老林给老张_评价模块.md`)

**总评**: 主线质量高, 无阻断. Provider 抽象跟 store_clone 完全对齐, 三表分离 (主/replies/settings) 职责清晰, UNIQUE KEY 防重 sync, 安全默认 (auto_reply 默认关 + 4 星才自动), Premium 订阅 graceful 失败沿用 search_insights 同款, Few-shot prompt 设计精致 (emoji 节制 + 俄语日常调性).

**3 条 Phase 1.3 优化建议** (按优先度):

1. 🟡 **Provider 每方法新建 client → lazy 缓存**: `wb.py:82` / `ozon.py:84,113,125` 每个方法都 `WBClient(...)` + `finally close()`. 跟 `store_clone/providers/ozon.py:OzonSellerProvider._get_client()` 同款抽 `_client` lazy 缓存.

2. 🟡 **`shop_review_replies` 加 `(review_id, sent_status='sent')` 唯一约束**: 防 race condition (用户狂点发送 + Celery 扫 pending 同时跑) 一评价多次发出. 用 generated column + UNIQUE 兜:
   ```sql
   ALTER TABLE shop_review_replies
     ADD COLUMN sent_lock_key VARCHAR(80)
       GENERATED ALWAYS AS (IF(sent_status='sent', CONCAT('sent-', review_id), NULL)) STORED,
     ADD UNIQUE KEY uk_one_sent_per_review (sent_lock_key);
   ```

3. 🟢 **`custom_prompt_extra` / `custom_hint` prompt injection 兜底 sanitize**: 黑名单关键词 (`ignore`/`system:`/`роль:` 等), 命中 → 拒收走默认 prompt. 优先级低 (店铺主自己用不是 attacker 场景).

**设计取舍待续 (不阻断)**: ENUM vs VARCHAR 一致性 / `product_id` 反查 (Ozon sku → platform_listings JOIN) / `auto_replied` 状态机进入条件.
