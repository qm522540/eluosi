## 一、AI 调价 v2 规则重构（核心，约 20 commit）

### 1. CTR/CR 取值规则重写
和用户做了完整的规则讨论（歧义 3 个 + 问题 4 个 + 意外 3 个），确认后一次性重构：

| 阶段 | CTR/CR 来源 | cpa_ratio | 保护机制 |
|---|---|---|---|
| Day 0 | shop_avg × 40%/20% | 0.50 | 保守系数 |
| Day 1-6 | growth_count 棘轮 | 0.50 | 保守系数 |
| Day 7-13 | 均值60% + 自身40% | 0.55 | 均值主导 |
| Day 14-20 | 自身70% + 均值30% | 0.58 | A'保护（偏离>50% + ROAS<保本线 → 回退60/40） |
| ≥21 | 纯自身 | 动态试探 | ROAS门控 + 利润试探 |

### 2. ROAS 门控（≥21天）
- last3 ROAS < 保本线 → 禁止加价
- last3 < prev3 × 0.9 → 加价砍半
- 降价直接放行

### 3. 利润试探（≥21天）
- `profit = spend × (ROAS × net_margin - 1)` 最大化
- 每3天评估 last3 vs prev3 利润，±0.05 调 cpa_ratio
- ad_groups 加 cpa_ratio / cpa_ratio_updated 字段（027 迁移）

### 4. 时段系数 + 星期系数
- 深夜 0.50、次高峰 1.10、晚高峰 1.20（缩小极差）
- 周五 1.05、周末 1.10

### 5. 出价取整
- 从 `round(x, 2)` 改为 `int(round(x))`

### 6. 亏损检测永远执行
- 不再受 auto_remove 开关控制
- auto_remove OFF → 写建议列表（用户确认）
- auto_execute 只管调价，auto_remove 只管删除

### 7. 平台最低出价校验
- WB：`POST /api/advert/v1/bids/min`（推荐竞争价，snake_case + placement_types 数组）
- Ozon：`GET /api/client/limits/list`（修了 _extract_ozon_min_bid 响应结构不匹配的 bug）
- 做法 C：预判阶段保留算法值不强拉，加警告"低于推荐价"，执行时走平台内部 fallback
- 降价反拉保护：想降价但 platform_min > current_bid → 跳过（避免"降价反涨价"）
- bid_adjustment_logs 写 actual_bid_rub（执行真实值，不再写建议值骗人）

### 8. expected_roas 修正
- 旧：client_price / bid（假设 CR=100%，错）
- 新：current_roas × current_bid / optimal_bid（实测比例换算，降价时 ROAS 正确升高）

### 9. "忽略/恢复" SKU 级开关
- 复用 ad_groups.user_managed 字段
- 新 API：`POST /suggestions/{id}/ignore` + `/restore`
- 被忽略 SKU 不参与自动调价/删除，但建议列表仍显示 AI 推荐
- 前端：忽略行灰色+🔒标签，只有"恢复"按钮，Tooltip 提示功能含义
