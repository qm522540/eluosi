## 五、今日体会

- **占位 onClick 要敢留**：商品管理里"新增商品 / 编辑 / 描述保存"都先留 `() => {}` / `message.success` 占位，先把列表 / 同步 / 铺货 / AI 改写的主干串起来。等后端接口定版再补细节，比一次性做完更省返工
- **平台 + 店铺合并下拉**比两个下拉体验好很多：原来用户先选平台再选店铺，容易选错组合；合并 OptGroup 后一次点击定位，也顺便天然保证 `shop_id` 参数不会漏传
- **失败弹窗要有细节**：AI 调价失败原来弹空白 modal，用户无法判断是 API 超时还是 quota 用完。加上具体 reason + 红色 × 后，上报问题变容易了
- **执行类按钮确认弹窗是标配**：AI 调价"立即执行"会真的改 WB/Ozon 出价，没有确认弹窗就是事故温床
- **部署前的 `git status`**：今天发现 Products.jsx 和 products.js 在我本地动手前就已经是 origin/main 最新版，避免了重复 Write。`git status` 永远是部署前第一步
