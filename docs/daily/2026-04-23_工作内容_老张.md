## 三、P1-A：`_calc_healthy_window_*` 加 platform 纵深过滤 `c729ff8`

### 3.1 背景

老林昨日 general-purpose agent 审查出的中危：两函数的 SQL 只按 `(tenant_id, campaign_id, ad_group_id)` 过滤，没带 platform。

### 3.2 生产验证

```sql
SELECT tenant_id, campaign_id, ad_group_id, COUNT(DISTINCT platform) pn
FROM ad_stats GROUP BY tenant_id, campaign_id, ad_group_id HAVING pn > 1
-- 结果：0 行
```

当前 `(tid,cid,agid)` 100% 隐含唯一 platform（因为 `ad_campaigns.id` 每行绑一个 platform，不跨平台共享 id）。**业务上不加 platform 也不会错**，但若 `ad_stats` 写入端出 bug 混入跨平台脏行，清洗结果会被污染 — 这是 agent 判中危的出发点。

### 3.3 改动

- `_calc_healthy_window_metrics` / `_calc_healthy_window_full` 新增 `platform: str | None = None` 参数
- SQL 动态拼 `AND platform=:platform`（只有传了 platform 才加）
- `_hill_climbing_decision` 顺道加 `platform` 参数，转发给内部 3 处调用
- `analyze_now` 两处调用点（line 1322 亏损建议、line 1400 hill 决策）传 `platform=platform`（循环中 `platform = shop.platform`）

默认 None 保向后兼容；调用方不传则退化为原行为（任何单测不需要改）。

### 3.4 合规自查

- 规则 1：SQL 里 `tenant_id` 原本就有 ✓
- 规则 4：不涉及手动触发接口
- 纵深防御思路同规则 1（每条 SQL 必带 tenant_id）

## 五、P1-C：WB 集群代表词黑名单 `6e72f25`

### 5.1 背景

昨日 B1：DeepSeek AI propose 阶段偶发非 WB 真实簇的代表词（`сережки с котиками золото` / `для девочки в подарок`）。这些词被 WB set-minus 测试"接受"（WB 接受 ≠ 真是集群代表），按原逻辑会持久化到 known_reps 污染下次查询。

### 5.2 设计

加 Redis 黑名单集合，`add_known_rep` 写入前检查，黑名单词直接 log + skip。

- Key: `wb:cluster_rep_blacklist:{advert_id}:{nm_id}`
- TTL：90 天（长于 known_reps 30 天，防过期后脏词复活）

### 5.3 改动

`app/services/ad/keyword_clustering.py`：
- 新增 `is_rep_blacklisted(adv, nm, rep) → bool`
- 新增 `add_rep_to_blacklist(adv, nm, rep) → int`：sadd 黑名单 + 同步从 known_reps 清除，返回清除数
- 新增 `remove_rep_from_blacklist(adv, nm, rep) → bool`：撤销误加
- 新增 `get_rep_blacklist(adv, nm) → List[str]`
- **改造 `add_known_rep`**：写入前检查黑名单，在则 `logger.info` 跳过

`app/api/v1/ads.py` 扩展 `POST /campaign-keywords/{id}/probe-cluster-rep`：
- Request 加 `action: str` 字段，默认 `"probe"`
- `action=blacklist`：直接拉黑 + 清 known_reps（不走 WB 验证）
- `action=unblacklist`：撤销
- `action=probe`：原行为
- 清 clusters 缓存抽成局部函数 `_bust_cluster_cache` 三 action 共用

### 5.4 生产 smoke test（6 场景全绿）

```
1. 初始: known_reps=[], blacklist=[]
2. add_known_rep(dirty) → known_reps=['сережки с котиками золото']
3. add_rep_to_blacklist(dirty) → removed=1, known_reps=[], blacklist=[dirty]
4. add_known_rep(dirty) 再次 → [log: 跳过黑名单] known_reps=[]
5. remove_rep_from_blacklist(dirty) → blacklist=[]
6. add_known_rep(dirty) 又能写 → known_reps=[dirty]
```
