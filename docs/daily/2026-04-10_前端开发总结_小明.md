## 二、详细技术实现

### 2.1 Ads.jsx 拆分（重构）

**背景：** 原 `src/pages/Ads.jsx` 单文件2117行，40+ useState，难以维护。

**拆分结果：**

```
src/pages/Ads.jsx (已废弃)
    ↓
src/pages/ads/
├── index.jsx              # 入口：Tab切换 + 店铺/平台选择器
├── AdsOverview.jsx        # Tab1：活动概览（列表/详情/广告组/关键词CRUD）
├── AdsAnalysis.jsx        # Tab4：数据分析（现改为ComingSoon占位）
├── AdsRules.jsx           # Tab2：自动化规则（4种规则管理）
├── AdsBudget.jsx          # Tab5：预算管理（现改为ComingSoon占位）
├── AdsAIPricing.jsx       # Tab3：AI智能调价（三平台统一）
└── ComingSoon.jsx         # 占位组件
```

**拆分原则：**
- 每个子组件只保留自己Tab相关的state和逻辑
- 公共的 `shopId`、`platform`、`searched` 由 index.jsx 通过props传入
- 所有API调用统一走 `src/api/ads.js`

**路由变更：** `App.jsx` 中 import 路径从 `@/pages/Ads` 改为 `@/pages/ads`（自动匹配 index.jsx）

#### 2.2.2 Ozon全自动模式（OzonAIPricing）

**功能区域：**

| 区域 | 功能 | 交互 |
|------|------|------|
| 策略模板配置 | 展示模板列表（标准/保守/激进/自定义） | Collapse默认折叠，点击展开编辑 |
| 模式开关 | 自动模式/建议模式切换 | Switch切换，调POST toggle-auto接口 |
| 待确认建议列表 | 商品级别出价建议（建议模式时显示） | 单条确认/忽略 + 批量操作 |
| 调价历史记录 | 已执行/已忽略/已过期记录 | Collapse折叠面板，支持日期筛选 |

**建议列表列设计（13列）：**

| 列 | 数据源 | 渲染逻辑 |
|----|--------|----------|
| 商品名称 | product_name + image_url | 缩略图+Ozon链接+活动名称 |
| 当前出价 | current_bid | ₽{整数} |
| 建议出价 | suggested_bid | 绿色加价/红色降价 |
| 调幅 | adjust_pct | +15%/-12% Tag |
| 数据质量 | campaign_data_days | Badge(充足/有限/不足) |
| 决策依据 | decision_basis | Tag(历史数据/店铺基准/预算控制) |
| 商品阶段 | product_stage | 蓝冷启动/橙测试/绿放量/红衰退 |
| 优化目标 | stage_optimize_target | 曝光量/CTR-CR/ROAS/利润 |
| 大促 | promo_phase | 预热/冲刺/恢复 Tag |
| ROAS | current_roas + expected_roas | 当前→预期，含颜色箭头 |
| AI理由 | reason | Tooltip展开完整内容 |
| 操作 | - | 执行/忽略按钮 |

**批量操作：** 使用 `Promise.allSettled` 实现，部分失败时显示成功/失败数量。

#### 2.2.3 WB建议模式（WBAIPricing）

**与Ozon的差异：**

| 维度 | Ozon | WB |
|------|------|----|
| 操作粒度 | 商品级别 | 活动级别 |
| 执行方式 | API自动执行 | 跳转WB后台手动改价 |
| 模式支持 | 自动+建议 | 仅建议 |
| 出价单位 | 卢布 | CPM |
| 操作按钮 | 执行/忽略 | 去WB改价（紫色#CB11AB）/忽略 |

**WB后台直链：** 优先使用 `record.wb_backend_url`，fallback到 `https://cmp.wildberries.ru/campaigns/list/active/edit/{campaign_id}`

**数据不足提示：** `data_days < 3` 的行加黄色背景（`.row-low-data { background: #fffbe6 }`）

#### 2.2.4 数据初始化检测

```
进入AI调价Tab
  ↓
GET /ai-pricing/data-status/{shopId}
  ↓
initialized=false → Spin + "正在拉取历史数据" + 每30秒轮询
initialized=true  → 正常渲染页面
接口报错           → fallback为已初始化，不阻塞
组件卸载           → clearInterval清除轮询
```
