## 五、今日体会

- **AI 异步长任务的 UX 三件套**：Spin tip 写明具体耗时（"5-15 秒"而不是"处理中"）+ 确认按钮 loading 状态 + **禁用 cancel/close/mask 点击**。用户没见过"15 秒"这么慢的按钮，不明确告知就会去点 X 或遮罩关窗，后端结果再返回也没用了
- **前端先判后端约束**：文档 §7.2 明说"没品类映射返 400"，就不要依赖后端 400 + message 让用户看——直接在弹窗里把 Radio 禁掉 + 整体 disable 提交按钮。用户根本不会撞到错
- **小步提交的信号灯效应**：今天一上来就开始写代码，每做完一档（菜单/API/骨架/Tree CRUD/品类 Tab/AI 推荐/属性 Tab/Drawer/打磨）立刻 commit + 先不 push，攒到最后一起推。每个 commit 都 build 过，任一步崩了直接 `git reset --hard HEAD~1` 就能回到上一档，没压力
- **组件拆文件夹比单文件多 900 行强**：一开始想把所有映射相关组件堆 `MappingManagement.jsx` 里，后来建 `pages/products/mapping/` 子目录拆成 6 个文件。每个 200-300 行，改一个组件只看一个文件，不用滚来滚去
- **禁用按钮 + tooltip 比灰态 + 文字说明更可发现**：属性值 AI 推荐按钮用的是 `disabled + Tooltip`，用户悬停会看到具体原因；如果只是文字说明写在旁边，很容易被忽略
