### 3.2 一开始 grep 零命中是误报

第一次跑：
```bash
grep -nE "datetime\.utcnow|datetime\.now\(\)|date\.today\(\)|\bNOW\(\)|UTC_TIMESTAMP|CURRENT_TIMESTAMP" app/tasks
```

→ **No matches found**

正则太严漏了 `datetime.now(timezone.utc)` 这种带参形式。规则 6 字面禁的是无 tz 的 `datetime.now()`，但**用 aware datetime 写 SQLAlchemy DateTime（无 tz）列也违反规则 6**——规则 6 第二段明确写："**唯一写入入口** `utc_now_naive()`"。

放宽到 `datetime\.now\b` 后命中 11 处。

| 文件 | 行号 | 字段 |
|---|---|---|
| `daily_stats.py` | 45, 59 | TaskLog.started_at / finished_at |
| `daily_sync_task.py` | 49, 64, 95 | TaskLog.started_at / finished_at（×2） |
| `roi_alert.py` | 58, 147, 167, 184 | TaskLog.started_at / finished_at（×2） + Notification.sent_at |
| `ad_auto_exclude_task.py` | 332, 379 | AdCampaignAutoExclude.last_run_at（×2） |

### 3.4 修法

- 4 个文件加 `utc_now_naive` 到 `from app.utils.moscow_time import` 行（`daily_sync_task.py` 没该 import 行，新加一行）
- 11 处 `datetime.now(timezone.utc)` → `utc_now_naive()`
- `from datetime import datetime, timezone` 行**保守保留**（避免误删未读完文件里的依赖；以后单独清理）

### 3.5 自查

```bash

grep -rn "\bNOW\(\)|CURRENT_TIMESTAMP|UTC_TIMESTAMP" app/services/ app/api/ app/tasks/  → 1 处 ads.py:2186 是注释行（说明文字）
```
