## 十四、今日体会

1. **小步提交是真正的安全网**：今天 25 个 commit，每个独立部署 + 端到端测过。中间被中断好几次（system-reminder / 用户插话），每次都能快速回到上下文，不丢任何工作。如果攒到最后一起推，今天的 WB 短语链 + 自动屏蔽 + Ozon 加起来一旦哪一步崩，整夜白干。

2. **WB API 的"看似生效"陷阱**：set-minus 用错字段名 (words vs norm_queries) WB 返回 200 OK 但实际不写入。如果不去 GET 验证写入结果，永远以为自己代码对。**所有写操作完都要 GET 一次校验**应该成为对接外部 API 的肌肉记忆。

3. **WB 短语限制要"治"在体验层**：底层 API 真的不支持，再怎么调也调不通。但可以从用户视角让这个限制"消失"——质检不出现这种词、表格标灰、message 措辞友好。**用户感受不到的限制 ≠ 限制**。

4. **基础数据资产复用是最高效的设计**：今天的"屏蔽规则" + "自动屏蔽托管" + "白名单" + "统一账本" 4 个新功能，全部复用了前期做的 efficiency_rules 表。一处改全栈生效，不需要为每个新功能新建一张配置表。**先建好基础设施再做上层应用**，远比"边做边加表"省力。

5. **Phase 拆分让"想得多但做得少"**：今天最大的"教训"——之前讨论 Ozon 关键词时一口气画了 3 个 Phase 的大饼，结果用户实际场景只需要 Phase A 一小块（而且还卡在 Premium）。**先 Phase A 落地，看实际数据再决定 B / C** 是更稳的节奏，避免一次性交付太多用户根本用不上的东西。

6. **决策点可视化是设计师的本职**：每次给用户多个方案（A/B/C）+ 工作量 + 价值对比表，用户决策更快，我也少踩"做错方向"的坑。今天至少 4 次"3 选 1"的明确决策点，效率比"我觉得这样好"高得多。
