## 三、用户截图揭开的真 bug：双 UI 开关语义错位

### 3.1 用户反馈

用户传截图：UI 显示「AI 智能调价：已开启」（绿字 + "AI 将自动调整出价，高峰 30 分钟 / 平稳 2 小时巡检"），期望 = 定时到点自动分析 + 自动改价。

**但 DB 显示**：shop=6 `is_active=0 + auto_execute=1` —— 矛盾状态。

### 3.2 链路诊断

反查文案所在文件 → `AdsAIPricing.jsx` line 1507（不是 `BidManagement.jsx`）。

```
前端 Switch onChange → handleToggleAuto(checked)
  → toggleAIAutoExecute(shopId, { auto_execute: checked })
  → PUT /bid-management/ai-pricing/configs/${shopId}
  → update_config (ai_pricing_executor.py)
  → 只更新 auto_execute 字段，is_active 从不被写入
```

**Celery `run_bid_management` 侧**：

```python
# app/tasks/bid_management.py _process_shop
SELECT is_active FROM ai_pricing_configs ...
if ai_active: dispatch execute(shop)
else: "未启用任何模式，跳过"
```

→ **Celery 只看 is_active**。用户点旧 UI 开关 → `auto_execute=1` 写入，`is_active=0` 不动 → Celery 判 is_active=0 跳过 → 什么都不发生。

**文案误导**："AI 将自动调整出价" 让用户以为开了就全自动，实际啥也没开。

### 3.3 方案 A hotfix `0a30a5b`

`AdsAIPricing.jsx` line 711 `handleToggleAuto` 开关改成：
- 开 → `enableAIPricing(shopId, true)` — 后端 `enable` 同时 `is_active=1 + auto_execute=1`
- 关 → `disableAIPricing(shopId)` — 同时两字段 0

`_flatten_template_rows` 补 `is_active` 字段给前端，`fetchConfigs` 按 `is_active AND auto_execute` 双真判 "已开启"。加 92003 冲突 Modal 指引。

部署后用户实测：关 → `is_active/auto_execute` 1→0；开 → 0→1。**双向干净切换** ✓
