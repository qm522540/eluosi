## 六、AI 标题三大输出 bug 4 层防御（晚上）

### 6.1 中文泄漏（如"环形"出现在俄语标题里）

根因：variant_attrs 里"型号: 环形" 中文值被 GLM dump 到标题。

4 层防御：
1. **输入侧过滤**：`_has_cjk(value)` 跳过含 CJK 的属性值，AI 收不到中文输入
2. **SYSTEM_PROMPT 硬规则 #7**：明确"禁止输出任何中文 / 拼音字符"
3. **后处理 `_strip_cjk(s)`**：正则去除所有 CJK Unicode 块
4. **失败兜底**：`prompt_dump` 完整入库 `keywords_used` JSON，便于复盘

### 6.2 重复词（如"серьги"出现 3 次）

根因：候选词包含 "серьги ракушки" / "серьги бабочки" / "серьги жемчуг" 这种带 серьги 前缀的近义短语，GLM 全部排列组合写进标题。

3 层防御：
1. **SYSTEM_PROMPT 规则 #8**："严格去重，同义词排列只算一次"
2. **SYSTEM_PROMPT 规则 #9**："禁用标点，token 间单空格"
3. **后处理 `_dedupe_tokens(s)`**：按 token 去重保序

### 6.3 跨品类窜词（如耳环标题里出现 "кольцо"）

根因：候选词没含 кольцо、原标题没含 кольцо、cat_top 也没勾，但 GLM 联想"耳环 = серьги + кольца(圆形)" 自己造了出来。

4 层防御：
1. **SYSTEM_PROMPT 规则 #10**："禁止跨品类词（耳环不加 кольцо/колье/брошь...）"
2. **后处理 `_strip_cross_category_tokens(s, current_cat_key)`** 自动剥离
3. **前端 preview 默认不勾选** looks_cross_category 词
4. **审计字段**：`cross_removed_by_post_check` 入库，能看 AI 哪几次违规

## 七、SEO 健康评分 3 个 bug 兜底（晚上）

### 7.1 `bindparam` UnboundLocalError 致命 bug（`2a8c614`）

`compute_shop_health` 在 `if type_ids:` 分支里写了 `from sqlalchemy import bindparam` 局部 import，把 bindparam 整个函数级变成局部变量。后面 line 348 的 `miss_stmt.bindparams(bindparam("pids", expanding=True))` 用到时，如果上面 if 分支没走，就触发 UnboundLocalError 整个 health 计算崩。

文件顶 line 29 已经全局 `from sqlalchemy import bindparam, text`，删掉局部 import 即可。

### 7.2 标题长度 30 字符正好踩 0 分边界

旧公式 `if n <= 50: sc = (n - 30) / 20 * 20`，n=30 时 `0/20*20 = 0`，标题正好 30 字符的商品（不少）直接判 0 分。

改：< 20 才 0 分；20-50 → `8 + (n-20)/30*12`（8→20 平滑递增），保证 20 字符就有 8 分起步。

### 7.3 coverage 评分双口径不一致

`_score_coverage(self ∪ cat_top)`：cat_top 都是 2+ 词俄语 phrase（如 "серьги кольца медицинский сплав"），30-50 字符短标题里子串匹配几乎全 miss → 0/30 → 0 分；但右侧"缺词 Top 3"列只看 self → 显示 "✓ 核心词已全部覆盖"。**两个口径完全矛盾**。

修：coverage 评分**只用 self**，跟"缺词 Top 3"完全一致。`self_total=0` 走 `data_insufficient=True` 豁免。`cat_top` 改 reference 信号 — detail 里附 "类目热门命中 X/Y"，纯展示不算分。前端 DimensionBar tooltip 同步改写。

## 十一、今日体会

1. **"用户的视觉直觉就是最准的 spec"**
   SS-E0349 / SS-E0383 那个截图，左边 "0/60" 右边 "✓ 核心词已全部覆盖" — 用户一眼就看出矛盾。我之前给 coverage 加 cat_top union 是想"覆盖率更全面"，但忘了"缺词 Top 3"列是只看 self 的，两边没同步逻辑。**任何评分 / 显示分两个口径，必出不一致**。

2. **"局部 import 改写函数级变量作用域"是 Python 经典坑**
   `from sqlalchemy import bindparam` 写在 `if type_ids:` 里面，整个函数体里 `bindparam` 全部变成局部变量。下游用到 `bindparam` 的代码 if 没走就崩。**Python 函数级 import 一律放函数顶或文件顶，不准嵌在条件分支里**。这事老林之前也念叨过。

3. **"AI 输出 4 层防御就够了，不要少层"**
   中文泄漏 / 重复 / 跨品类窜词三个 bug 各自 3-4 层防御。少一层都会偶发漏。**SYSTEM_PROMPT 不能信，post-check 必须有，输入侧也要洗干净**。今天用户连续报这三个 bug，每个都是"前两层做了但第三/四层没做"被打穿。

4. **"用户每条数据自决 > 系统自动推荐"**
   AI 生成相关的输入字段全改成可勾选，看起来"麻烦了"，但用户实际反馈"舒服多了"。原因：之前是黑盒"AI 帮你想"，现在是白盒"AI 用你给的料"。**AI 工具的可信感来自可控感**。

5. **"Ozon 写回成功标志在 task_id 落库，但实际生效要等 1-5 分钟"**
   今天 Apply 实测拿到 task_id 后，前端立刻显示"已提交"。但用户去 Ozon 后台看可能要 5 分钟才同步。**异步任务的 UX 必须把"已提交 ≠ 已生效"明示**，否则用户秒去看后台没变就以为是 bug。
