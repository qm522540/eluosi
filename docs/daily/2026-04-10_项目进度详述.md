## 三、已完成模块详述

### 3.1 系统认证与权限

- 用户注册/登录（JWT Token，24小时过期）
- 多租户隔离（tenant_id贯穿全表）
- 角色区分（owner/admin/member）
- 个人信息修改（用户名/邮箱/密码）

### 3.2 店铺管理

- 支持三平台：Wildberries (WB) / Ozon / Yandex Market
- 店铺CRUD + API凭证管理
- 连接测试（验证API Key有效性）
- Ozon双凭证体系：
  - Seller API（Client-Id + API Key）：商品、订单
  - Performance API（OAuth Client ID + Secret）：广告投放

### 3.3 广告管理（重点模块）

#### 3.3.1 活动概览
- 活动列表（支持平台/店铺/状态筛选）
- 活动状态管理（active/paused/archived）
- 从平台API同步活动数据
- Ozon活动详情：预算余额卡片 + 商品出价列表 + 出价在线修改

#### 3.3.2 数据分析
- **平台对比分析**：跨平台花费/收入/ROAS/转化率对比图表
- **活动TOP排名**：按花费/收入/点击/订单多维排序
- **商品级ROI分析**：通过广告组关联listing_id，分析每个商品的ROAS/CPA

#### 3.3.3 自动化规则引擎（4种规则）

| 规则类型 | 说明 | 执行方式 |
|----------|------|----------|
| 低ROI自动暂停 | ROAS低于阈值且花费超限时暂停活动 | 本地数据库状态修改 |
| 分时自动调价 | 按莫斯科时间4时段调整出价比例 | **Ozon: 直接调Performance API** |
| 预算封顶 | 日花费超限自动暂停活动 | 本地数据库状态修改 |
| 库存联动 | 库存不足暂停/库存恢复开启 | 预留（等库存模块对接） |

**分时调价特性（4/10重点改造）：**
- Ozon平台直接通过Performance API读取商品出价 → 计算调价 → 调用API修改
- 非Ozon平台沿用本地AdGroup调价逻辑
- 出价取整为整数卢布（Ozon最低3卢布限制）
- 首次运行记录原始出价，防止叠加调价
- 关闭/删除规则时自动恢复原始出价（弹窗确认）
- 启用状态下禁止编辑规则
- 三时段小时选择器互斥（防止冲突）
- 每次调价写入AdBidLog日志
- Celery Beat每小时25分自动执行
- 支持手动"立即执行"

**规则管理特性：**
- 按店铺级别隔离，同店铺同类型只能一条
- 类型即名称，编辑时不可改类型
- 执行结果返回详细调试信息（moscow_hour/period/adjust_pct/产品详情/API响应）

#### 3.3.4 预算管理
- **消耗概览**：总日预算、今日花费、本月花费、预算使用率
- **预算预警**：使用率>=80%标橙预警，>=100%标红
- **活动明细**：每个活动的消耗进度条、均日消耗、剩余天数
- **优化建议**：基于7天ROAS自动生成加/降预算建议

#### 3.3.5 页面交互优化（4/10）
- 4个Tab（概览/数据分析/自动化规则/预算管理）始终可见
- 内容按店铺维度管理，选择店铺确定后加载
- 隐藏暂不需要的"新建活动"和"告警"按钮
- 移除广告趋势卡片和活动列表的费用/展现次数列

### 3.4 数据采集系统

| 平台 | 采集内容 | 频率 |
|------|----------|------|
| WB | 广告活动列表、预算 | 每小时05分 |
| Ozon | 广告活动列表、商品出价、统计数据 | 每小时10分 |
| Yandex | 广告活动列表、统计数据 | 每小时15分 |

**日统计任务**：每日0/1/2点采集三平台订单、收入、退款等数据

### 3.5 系统设置
- 个人信息（用户名/邮箱修改）
- 密码修改（旧密码验证）
- 通知偏好配置
- 企业微信机器人配置（webhook）
