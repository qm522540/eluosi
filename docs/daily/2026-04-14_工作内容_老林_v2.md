## 二、调研与讨论（非 commit 但重要）

### A. WB / Ozon 平台最低出价 API（已实测）

两平台都**有接口可用**，两份 reference memory 已入 `~/.claude/projects/D--eluosi/memory/`：

- **WB**: `POST /api/advert/v1/bids/min`，按 活动ID+nm_id+payment_type+placement 查询。实测 SKU 531721775 在活动 33121342 下最低 CPM = 8000 kopecks = 80₽。**字段坑点**：body 必须 snake_case，payment_type 必须小写 `"cpm"`，返回值单位 kopecks 要 ÷100
- **Ozon**: `GET /api/client/limits/list`，返回 7 组限制（objectType × paymentMethod × placement × category）。SKU+CPC 最低 7-10₽，全局最高 100-200₽，每组带 281 品类的专属 minBid

### B. AI 调价机制完整梳理（对话产出，未归档）

给用户完整讲解了：触发方式、数据分段（last5/prev5/week2-4）、prompt 结构、guardrail 链、安全护栏、建议写入/执行、平台调价 API、日志追溯。

核心原则一句话：**"数据够才动、降价比加价宽松、ROAS 绝对值优先于趋势方向、小步调整（5-10%）"**

### C. 算法方向待决策（入 memory）

用户提出：是否把"增量调整"改为"绝对值推导"（unit economics 反推理论最优 CPM）。

- 用 895200249 实算：target_cpc = 客单价 × 毛利 / target_roas = 525 × 0.5 / 3 = 87.5₽；理论 CPM = 87.5 × 3% × 1000 = 2625₽。当前 bid 45₽，差 52 倍
- 未决策。存入 `project_ai_pricing_algorithm_direction.md` 备后续讨论

### D. current_bid 来源说明（入 memory）

回答用户"改过一次价后第二次 AI 决策的原价是啥"：每次都实时从平台 API 拉当前出价，不缓存不累积。与用户手动改价共存。存入 `project_ai_pricing_current_bid_source.md`。

### E. 商品售价接入路径（未实施）

实测 WB `discounts-prices-api.wildberries.ru/api/v2/list/goods/filter` 可拿：

- 895200249：原价 2100₽、折后 525₽、打 2.5 折 75% off
- 历史"客单价 525" = discountedPrice（不是巧合，每单 1 件）
- **发现**：打折力度大的商品真实毛利率可能只有 20%，用默认 `margin=0.5` 算的 target_cpc 过于乐观

### F. CPM 数据接入（差一行代码）

用户问"更新数据源时能不能拉到 CPM"。答：`_calc_metrics` 加一行 `cpm = spend / impressions × 1000` 就有。WB 场景非常有用（平台就是 CPM 计费），Ozon 场景参考价值低（多为 CPC 计费）。

### G. ROAS 合理目标讨论

用户问"毛利 60% 的 ROAS 目标合理值"。给建议：
- 保本 ROAS = 1/0.6 ≈ 1.67（但没考虑平台佣金 15-23% + 物流 + 退货）
- **推荐**：`target_roas=2.5, min_roas=2.0`（稳健）
- 激进扩量：2.0/1.7；稳定高利润：3.0/2.3

### H. 建议删除按钮触发条件说明

条件（三个同时满足）：
1. `product_stage === 'declining'`
2. `data_days >= 7`
3. `current_roas == null || current_roas < min_roas(默认1.8)`

点击后调平台 API 把 SKU 从活动里移除（不是删商品，是停止投放）。

### I. user_managed 字段机制说明

用户手动接管某个 SKU 出价后标记 `ad_groups.user_managed=true`，**定时 AI 调价跳过**，但用户在 WB/Ozon 原生后台改价不会自动标记（项目感知不到外部改动）。

### J. 出价管理诊断"全量决策 0"说明

当前店铺 6 个 SKU 全部未达 10 天数据，才出现这个状态。唯一有 7 天数据的是 895200249，只能走"仅可降价"档。其他 5 个在攒数据中。

### K. 895200249 降价判断分析

实算 last5 ROAS=3.42 vs prev5 ROAS=8.38 触发 trend=down，AI 按规则 5 给降价 5-10%。**但 prev5 其实是 04-07 一天爆发（占 80% 收入）拉高的假象**，订单稀疏场景下"趋势"判断不可靠。规则本身局限，数据窗口需加长。
