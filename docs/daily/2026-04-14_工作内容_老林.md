### 2. guardrail 风险不对称调整

原设计 `days<10` 全拦，导致有 ROAS 明显过低需要降价止损的 SKU 也被拦（data=7 天没机会调）。改为：

- **days<7**：全丢（数据太少不信）
- **days 7-9**：**仅允许降价**（止损是保护，风险小）
- **days>=10**：正常决策（加价需 trend 非 down）
- **trend=new**：全丢

prompt 硬规则区同步从 3 条扩成 4 条（A/B/C/D）。

### 3. prompt 注入当前莫斯科时段作为次要权重

### 10. WB + Ozon 平台最低出价 API 调研

用户问"AI 调价时能不能拿到平台商品最低出价"。查了两个平台：

- **WB**：`POST /api/advert/v1/bids/min` 可用。按 `活动ID + nm_id + payment_type + placement` 返回精确最低值。实测 SKU 531721775 在活动 33121342 下最低 CPM = 8000 kopecks (80₽)。重要坑点：body 必须 snake_case，payment_type 必须小写 `"cpm"`，返回值单位 kopecks 要 ÷100
- **Ozon**：`GET /api/client/limits/list` 返回 7 组限制（`objectType × paymentMethod × placement × category`），每组带 281 品类的专属 minBid。SKU+CPC 最低 7-10₽，全局最高 100-200₽

两份 reference memory 已写入 `~/.claude/projects/D--eluosi/memory/`，以备接入。
