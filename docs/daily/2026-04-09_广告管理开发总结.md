## 二、新增功能（5大模块）

### 1. 数据分析增强
- **平台对比分析**：跨平台花费/收入/ROAS/转化率对比（含图表）
- **活动TOP排名**：按花费/收入/点击/订单多维排序
- **商品级ROI分析**：通过广告组关联listing_id，分析每个商品的ROAS/CPA

### 2. 自动化规则引擎
- **低ROI自动暂停**：ROAS低于阈值且花费超过最低金额时自动暂停
- **分时自动调价**：4个时段（高峰/次高峰/低谷/平峰）分别设置出价加减比例
  - 基于莫斯科时间自动判断时段
  - 记录原始出价防止叠加
  - 每次调价写入日志（调价报表）
- **预算封顶**：日花费超限自动暂停活动
- **库存联动**：库存不足暂停、库存恢复开启（执行逻辑预留，等库存模块对接）
- 规则按**店铺级别**隔离，同店铺同类型只能一条

### 3. 预算管理
- **消耗概览**：总日预算、今日花费、本月花费、预算使用率
- **预算预警**：使用率>=80%自动预警，>=100%标红
- **活动明细**：每个活动的消耗进度条、均日消耗、剩余天数
- **优化建议**：基于7天ROAS自动生成加/降预算建议

### 4. 活动详情页增强（Ozon）
- **预算余额卡片**：绿色区域高亮显示预算余额
- **商品出价列表**：展示活动关联的商品SKU、名称、出价
- **出价在线修改**：点击出价数字可修改，同步到Ozon后台（已验证生效）
  - 正确接口：`PUT /api/client/campaign/{id}/products` + `{"bids":[...]}`
  - 单位换算：API微单位 / 1000000 = 卢布

### 5. 调价日志（报表）
- 记录每次调价的时间、广告ID、原出价、新出价、调幅、原因
- 在每条分时调价规则内通过"日志"按钮弹窗查看

## 三、平台对接

### Wildberries (WB)
| 接口 | 状态 | 说明 |
|------|------|------|
| `/adv/v1/promotion/count` | 可用 | 活动列表（ID、状态、类型） |
| `/adv/v1/budget?id=X` | 可用 | 预算余额 |
| `/adv/v1/promotion/adverts` | 404 已移除 | 活动详情/名称（WB已关闭） |
| `/adv/v2/fullstats` | 404 已移除 | 统计数据 |
| `/adv/v3/fullstats` | 参数问题 | 需要ids+日期，格式待确认 |
| 商品列表/出价修改 | 不可用 | 所有相关接口均404 |

**WB状态码修正：**
- status 9 = 活跃（统一广告/CPM），非暂停
- type 8/9 = 竞价广告(CPM/Auction)，新增`auction`广告类型

**WB已知限制：**
- 活动名称无法从API获取，需用户手动编辑
- 商品列表/出价无法通过API获取或修改

### Ozon
| 接口 | 状态 | 说明 |
|------|------|------|
| Performance API OAuth | 可用 | `POST /api/client/token` 获取Bearer Token |
| `GET /api/client/campaign` | 可用 | 活动列表（含名称、状态、类型） |
| `GET /api/client/campaign/{id}/v2/products` | 可用 | 商品列表+出价 |
| `PUT /api/client/campaign/{id}/products` | 可用 | 修改出价 `{"bids":[...]}` |
| `GET /api/client/campaign/{id}/objects` | 可用 | 商品ID列表 |
| Seller API 商品接口 | 全部404 | 无法获取商品图片 |

**Ozon关键发现：**
- 卖家API和广告API是两套独立认证系统
- 广告API需要Performance API的OAuth凭证（client_id + client_secret）
- 出价单位：API返回微单位，除以1000000 = 卢布
- 状态新增映射：`CAMPAIGN_STATE_FINISHED` = archived

### 店铺设置更新
- Ozon店铺支持**双凭证**配置：
  - 卖家API：Client ID + API Key（商品/订单/库存）
  - 广告API：Performance Client ID + Client Secret（广告数据）
- 各平台API Key输入框均添加了获取路径说明

## 八、部署注意事项

1. 执行5个数据库迁移SQL（011-015）

3. 重建前端：`cd frontend && npm run build`
4. Ozon店铺需配置Performance API凭证才能同步广告数据
5. WB活动名称需用户手动在编辑弹窗中设置
