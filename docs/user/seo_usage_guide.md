# SEO 模块用户操作指南

> 面向：店铺运营 / 老板
> 目标：**用 6 步把 AI 反哺出的搜索词变成真能吃流量的商品标题，并用数据验证效果**
> 覆盖版本：一期 MVP（2026-04-20）+ Tracking / ROI 报表（2026-04-21）

---

## 0. 这个模块在解决什么问题

你的商品 **被用户搜到了**，但因为标题里没写那个词，**平台没给你展示**。
SEO 模块做四件事：

1. **找词**：从付费广告 + 自然搜索词里挖"用户真在搜 + 你真有订单 + 你标题还没写"的词
2. **评分**：算出每个商品 SEO 健康度 0-100 分，告诉你该先改哪一个
3. **生成**：用 AI 把反哺词融合进你原标题，输出一个新俄语标题
4. **验证**：关键词表现追踪（环比看哪些词在掉）+ Before/After ROI 对比（改完真有没有涨）

你做的只有：**挑 3-5 个差商品 → 点 AI 生成 → 复制到平台后台 → 标记已用 → 14 天后回来看 ROI**。

## 1. 前置准备

### 1.1 商品数据已同步

- 店铺设置里 API 凭证齐全（Ozon 需 client_id + api_key，WB 需 api_key）
- 商品列表能看到商品 → 代表 sync 过了

### 1.2 搜索词数据有货

打开 **"数据报表 → 搜索词洞察"**，选店铺点"手动同步"。

- **OZON-Shario**：需要店铺开了 **Ozon Premium 订阅**，否则 API 返 403
- **WB-Shario**：需要店铺开了 **WB Jam 订阅**，否则 API 返 403
- 同步成功会显示"写入 N 条"，N ≥ 50 基本够用

### 1.3 跑一次 SEO 引擎

打开 **"SEO 管理 → 优化建议"**，顶部选店铺，点右上角「**刷新引擎**」。

- 引擎会扫描付费 + 自然两路数据，聚合出**反哺候选词池**
- 跑完会在右上显示"扫描 N 对 / 候选 M 条 / 入库 K 条"
- 如果 K = 0 可以：调低 ROAS 阈值（默认 2.0 → 1.5）；确认店铺有广告或搜索词历史

## 2. 核心 6 步闭环

### Step 1 — 首页看"SEO 健康概览"（两个入口）

路由：`/`（Dashboard）

- 滚动到页面最下方「**SEO 健康概览**」卡片
- 每店铺一张卡：
  - **健康均分**（红 < 40 / 黄 40-70 / 绿 ≥ 70）+ 差中优分布 + 最差 Top 3 商品
  - **关键词追踪摘要**：`⚠️ N 下滑 / 🌱 N 新增`（需店铺搜索词洞察有数据）
- 两个入口按钮：
  - 「去 SEO 健康诊断 →」→ 跳 `/seo/health`（Step 2）
  - 「去关键词追踪 →」→ 跳 `/seo/tracking`（Step 2.5，可选）

> 💡 **怎么读**：
> - 整店平均 < 40 → 大盘候选词覆盖率普遍偏低，优先改"最差 Top 3"
> - 下滑数 > 0 → 有核心词曝光断崖，先去 Tracking 排查可能是广告停投或平台算法变化
> - `数据未就绪` 灰字 → 搜索词洞察订阅未开通（见 §1.2）

### Step 2 — "SEO 健康诊断"挑待改的商品

路由：`/seo/health`（从 Dashboard 跳来会自动选中店铺）

- 表格按**分数升序**（最差在前），找红色分数的商品
- 每行 3 条进度条告诉你扣分在哪：
  - **关键词覆盖率** 60% —— 反哺词有多少已经写进标题/属性
  - **标题长度** 20% —— 30-180 字符为理想区间
  - **评分** 20% —— 平台商品评分（Ozon 暂无数据 → 自动豁免）
- 每行右侧有「**缺的高价值词 Top 3**」—— 红色 Tag 显示具体哪些词没写
- 操作列点 **[AI 优化标题]** → 自动跳去 Step 3 并自动选好 Top 5 候选词弹窗

> 💡 **怎么读**：
> - 进度条"覆盖率"红色 0% = 反哺词一个都没写进标题，改标题收益最大
> - "无数据·已豁免"灰色 tag = 这个维度没数据（如 Ozon 评分），不算扣分

### Step 3 — "SEO 优化建议"勾词 + AI 生成标题

路由：`/seo/optimize`（从 Health 跳来会自动过滤到单商品 + 弹出 AI 生成窗口）

#### 手动流程（不是从 Health 跳来的情况）

1. 顶部选店铺 → 表格出现该店所有候选词（通常 1000+ 条）
2. 筛选器建议：
   - **来源** = `自然·本商品`（最准，用户真搜真下单）
   - **状态** = `pending`（未处理）
   - **覆盖** = `未覆盖`
3. **勾选同一个商品的 3-8 个词**（建议别超 10 个）
4. 底部浮动工具条点 **[AI 生成标题]** 按钮

#### AI 生成窗口

- 打开时展示：商品信息 + 选中候选词 Tag + 原俄语标题
- 点「开始生成」→ AI 大约 **4 秒**返新标题
- 结果展示：
  - 绿色 Alert 显示**新俄语标题**（字号 15px + 一键复制）
  - 淡蓝元数据卡：AI 用词 Tag + 模型/耗时/Token
- 按钮：
  - **[一键复制新标题]** → 复制到剪贴板
  - **[重新生成]** → 再跑一次（AI 输出有随机性）
  - **[关闭]**

> ⚠️ **跨商品提醒**：勾选的词必须属于**同一个商品**。勾跨商品时会弹提示"涉及 N 个不同商品，AI 只能针对单个商品"。

### Step 4 — 去平台后台改标题

**当前一期不做"一键写回商品"**（三期计划）。流程：

1. 复制新标题
2. 打开 Ozon 卖家后台 / WB 卖家后台，找到对应商品
3. 粘贴到"商品名称"字段保存
4. 平台生效通常 < 1 小时

> 💡 提示：改之前**把旧标题复制一份**存在 Excel，万一效果不好可以回滚。

### Step 5 — 标记已应用 + 等效果

路由：`/seo/report`（菜单"SEO 管理 → 效果报表"）

- 这里能看到你 AI 生成过的**所有标题历史**
- 你去平台改完后，**回来点 [标记已用]**
- 这一步很重要 —— 二期 Before/After ROI 对比会用 `applied_at` 作为时间基线
  - 观察期：改完 14 天
  - 观察指标：商品曝光、订单、ROAS、自然搜索排名

## 3. 常见问题 FAQ

### Q1: 候选池是空的，怎么办？

可能原因：
- **订阅未开**：Ozon Premium / WB Jam 未开 → 搜索词洞察页同步不到数据
- **店铺太新**：没广告历史 + 没搜索曝光 → 没数据可反哺
- **引擎没跑**：点"刷新引擎"跑一次

调低 ROAS 阈值（2.0 → 1.5 → 1.0）再试，放大候选池。

### Q2: AI 生成的标题我不喜欢 / 丢了品牌名

- 点「**重新生成**」—— AI 有随机性，多跑几次会更好
- **系统提示词会要求 AI 保留品牌**，但偶尔会丢失。丢失时回到商品列表确认 `brand` 字段有值
- 手动改：复制后在平台后台编辑时自己加回品牌名

### Q3: Health 页全店 78 个都是"差"怎么办？

这不是 bug 是真相：
- 全店候选词覆盖率普遍低 = 整店 SEO 基础薄弱
- **策略**：不用一次全改，每周选 3-5 个"最差且订单多"的商品集中优化
- 14 天后看这批商品的曝光/订单增量，确认效果后复制到下一批

### Q4: 改完标题要等多久看效果？

- **平台索引**：< 1 小时
- **曝光变化**：3-7 天（搜索引擎要重新爬 + 排名稳定）
- **订单影响**：7-14 天（用户看到 + 下单周期）
- **推荐观察期**：14 天

### Q5: AI 生成需要花钱吗？

- 模型：GLM-4-Flash（智谱清言）
- **单次约 ¥0.06**（500-700 token）
- 全店 2000 条候选词全跑一遍约 ¥5 级别
- 记录在 `ai_decision_logs` 表，用量可审计

### Q6: 我已经改过标题，怎么让系统知道？

去 `/seo/report` 找到对应记录 → 点 **[标记已用]** → 状态变成 `applied`。

或者打开「详情」：如果"当前标题 = 生成标题" → 系统自动识别"疑似已应用"。

## 4. 故障排查

### 生成按钮 disabled

- **未选行**：先勾几行候选词
- **跨商品**：检查勾选的行是否属于同一 product_id

### 生成后 Modal 报错"AI 调用失败"

- 后端日志查 `seo_generation` 任务记录
- 常见原因：GLM_API_KEY 未配置 / 网络超时 / 上下文超限（候选词 > 30 个）
- 重试：关闭弹窗 → 重选 ≤ 10 个词 → 再试

### Health 页分数全 0

- 商品 listing 表 title_ru 为空 + 候选池为空 → 三维全 0
- 解法：先跑 Step 1-3 前置（同步商品 + 刷新引擎）

### Dashboard SEO 卡片一直转圈

- 后端 `/seo/shop/{id}/health` 接口 timeout → 查后端日志
- 可能店铺候选池过大（> 10000 条），SQL 聚合慢 → 联系开发加索引
